About VM Assignment Portal

VM Assignment Portal (VM AP) is a web-based application designed to simplify the management of virtual machines within a team. The application is built on Node.js and Angular 13, providing a more enhanced user interface with advanced features.

VM AP allows for the creation of VM entries, after which various operations like assign, release, and snapshot management can be performed. The portal is integrated with vCenter to get VM information and perform actions like taking and reverting snapshots. VMs of one team are not visible to another team, and all team leaders, users, and VMs are mapped with their team.

This code includes a hero section with a background image and a call to action button to get started. You can customize the text and button to fit your needs. Additionally, the code includes links to Bootstrap 5 CSS and JS, which are used to style the page and add interactive features.

Comprehensive Introduction to VM Assignment Portal

VM Assignment Portal (VM AP) is a web application that enables teams to easily manage their virtual machine (VM) related activities. It allows users to create VM entries and perform various operations such as assigning and releasing VMs. The application is integrated with VCenter to get VM information and perform actions like taking and reverting snapshots.

Roles in VM Assignment Portal

VM Assignment Portal has three roles:

  1. Admin
  2. Team Leader
  3. User

The Admin creates Team and Team Leader accounts. The Team Leader can create Users and add VMs. Almost all basic operations of the VM Assignment Portal can be performed by the User.

Team Leader

The Team Leader can manage various VM Assignment Portal activities at the team level. In case a new VM is required, the Team Leader sends a request to their manager. It is the responsibility of the Team Leader to monitor snapshot count and keep them less than 6 in all VMs.

The following actions can be performed by the Team Leader:

  • Create User
  • Edit User
  • Delete User
  • Remove VM
  • Sync Snapshot Count
  • Sync VM Additional Data
  • Activity Logs

The Snapshot Count of all the VMs of the team can be synced from VCenter. The Additional Data of all the VMs of the team can be synced from VCenter.

User

The User can perform various operations, including adding, assigning, releasing, updating VMs, adding comments, deleting comments, opening ICD, opening SP, Linux Tool Box, checking ICD Upgrade Status, DB Tool Box, snapshot management, and adding additional information. Almost all basic operations of the VM Assignment Portal can be performed by the User.

When a new VM is required, Users shall contact their Team Leader. The following operations can be performed by the User:

  • Add VM
  • Assign VM
  • Release VM
  • Update VM
  • Add Comment
  • Delete Comment
  • Open ICD
  • Open SP
  • Linux Tool Box
  • Check ICD Upgrade Status
  • DB Tool Box
  • Snapshot Management
  • Add Additional Info

Various snapshot related operations can be performed from the VM Assignment Portal. A new snapshot can be taken by clicking on the 'Take Snapshot' button. Users should not keep more than 5 snapshots on any VM (as per IBM project guidelines). The VM can be reverted to any existing snapshot. The 'View All Snapshots' option shall fetch the list of all snapshots taken on all VMs of the team. The 'Search Snapshots' option shall fetch a list of all snapshots having searched keyword in their Name or Description.
